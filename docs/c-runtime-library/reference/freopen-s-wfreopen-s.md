# <a name="freopen_s-_wfreopen_s"></a>`freopen_s`, `_wfreopen_s`

Закрывает файл, который в данный момент связан с `oldStream` , и переназначает его `stream` файлу, указанному параметром `fileName` .

Эти версии [`freopen, _wfreopen`](freopen-wfreopen.md) имеют улучшения системы безопасности, как описано в разделе [функции безопасности в CRT](../../c-runtime-library/security-features-in-the-crt.md).

## <a name="syntax"></a>Синтаксис

```C
errno_t freopen_s(
   FILE ** stream,
   const char * fileName,
   const char * mode,
   FILE* oldStream
);

errno_t _wfreopen_s(
   FILE ** stream,
   const wchar_t * fileName,
   const wchar_t * mode,
   FILE * oldStream
);
```

### <a name="parameters"></a>Параметры

*`stream`*\
Выходной параметр, который будет указывать на повторно открытый поток при возврате функции.

*`fileName`*\
Путь к файлу, который необходимо повторно открыть.

*` mode`*\
Режим для повторно открытого потока.

*`oldStream`*\
Поток, который необходимо открыть повторно. Эта запись закрывается, и все связанные с ней файлы закрываются.

## <a name="return-value"></a>Возвращаемое значение

Ноль при успешном выполнении; в противном случае — код ошибки. При возникновении ошибки исходный файл закрывается и **`NULL`** записывается в, *`stream`* Если *`stream`* не указано также **`NULL`**

Дополнительные сведения о кодах ошибок см. в разделе [`errno, _doserrno, _sys_errlist, and _sys_nerr`](../../c-runtime-library/errno-doserrno-sys-errlist-and-sys-nerr.md) .

## <a name="remarks"></a>Комментарии

**`freopen_s`** Функция обычно используется для присоединения предварительно открытых потоков, связанных с `stdin` , `stdout` и `stderr` в другой файл.

**`freopen_s`** Функция закрывает файл, который в данный момент связан с, *`stream`* и переназначает его *`stream`* файлу , указанному путем. **`_wfreopen_s`** — Это версия с расширенными символами **`freopen_s`** ; *путь* и *` mode`* аргументы для **`_wfreopen_s`** строк расширенных символов. **`_wfreopen_s`****`freopen_s`** поведение и идентично в противном случае.

Если какой-либо из *PFILE*, *path*, *` mode`* или *`stream`* **`NULL`** имеет значение, или если *path* является пустой строкой, эти функции вызывают обработчик недопустимых параметров, как описано в разделе [Проверка параметров](../../c-runtime-library/parameter-validation.md). Если выполнение может быть продолжено, эти функции устанавливают **`errno`** в значение **`EINVAL`** и возвращают **`EINVAL`** .

|---------------------|------------------------------------|--------------------|-----------------------|
|**`_tfreopen_s`**|**`freopen_s`**|**`freopen_s`**|**`_wfreopen_s`**|

**`freopen_s`** обычно используется для перенаправления предварительно открытых файлов **`stdin`** , **`stdout`** и **`stderr`** в файлы, указанные пользователем. Новый файл, связанный с *`stream`* , открывается с помощью *` mode`* , который представляет собой символьную строку, указывающую тип доступа, запрошенный для файла, следующим образом:

|*` mode`*|Access|
|-|-|
| **`"r"`** | Открывает для чтения. Если файл не существует или не найден, **`freopen_s`** вызов завершается ошибкой. |
| **`"w"`** | Открывает пустой файл для записи. Если указанный файл существует, его содержимое удаляется. |
| **`"a"`** | Открывается для записи в конце файла (добавление) без удаления маркера в конце файла (EOF) перед записью новых данных в файл. Создает файл, если он не существует. |
| **`"r+"`** | Открывает для чтения и записи. Файл должен существовать. |
| **`"w+"`** | Открывает пустой файл для чтения и записи. Если файл существует, его содержимое удаляется. |
| **`"a+"`** | Открывается для чтения и добавления. Операция добавления включает удаления маркера EOF перед записью новых данных в файл. Маркер EOF не восстанавливается после завершения записи. Создает файл, если он не существует. |

Используйте **`"w"`** типы и **`"w+"`** с осторожностью, так как они могут уничтожить существующие файлы. Начиная с C11 можно добавить **`"x"`** в или, **`"w"`** **`"w+"`** чтобы функция завершилась ошибкой, если файл существует, вместо перезаписи.

Когда файл открывается с **`"a"`** **`"a+"`** типом доступа или, все операции записи выполняются в конце файла. Хотя указатель файла можно перемещать с помощью [`fseek`](fseek-fseeki64.md) или [`rewind`](rewind.md) , указатель файла всегда перемещается обратно в конец файла перед выполнением любой операции записи. Поэтому существующие данные не могут быть перезаписаны.

**`"a"`** Перед добавлением в файл режим не удаляет маркер конца файла. После добавления команда MS-DOS TYPE отображает данные только до первоначального маркера EOF и не отображает данные, добавленные в файл. **`"a+"`** Перед добавлением в файл режим удаляет маркер EOF. После добавления команда TYPE MS-DOS отображает все данные в файле. **`"a+"`** Режим необходим для добавления в потоковый файл, заканчивающийся маркером конца файла CTRL + Z.

Если **`"r+"`** **`"w+"`** **`"a+"`** указан тип доступа, или, то чтение и запись разрешены (файл считается открытым для обновления). Однако при переключении между чтением и записью должны существовать промежуточные [`fsetpos`](fsetpos.md) [`fseek`](fseek-fseeki64.md) операции, или [`rewind`](rewind.md) . При необходимости можно указать текущую точку для [`fsetpos`](fsetpos.md) операции или [`fseek`](fseek-fseeki64.md) . В дополнение к приведенным выше значениям в строку могут быть добавлены один из следующих символов *` mode`* для указания режима преобразования для новых строк.

|*` mode`* Модификатор|Режим преобразования|
|-|-|
| **`t`** | Откройте файл в текстовом (переведенном) режиме. |
| **`b`** | Открыть в двоичном (непреобразованном) режиме; переводы, включающие символы возврата каретки и перевода строки, подавляются. |

В текстовом режиме (преобразованном) сочетания символов возврата каретки и перевода строки (CR-LF) преобразуются в символы перевода строки (LF) на вход. Символы LF преобразуются в комбинации CR-LF на выходе. Кроме того, при вводе символ CTRL+Z интерпретируется как символ конца файла. В файлах, открытых для чтения или для записи и чтения с помощью **`"a+"`** , Библиотека времени выполнения проверяет CTRL + Z в конце файла и удаляет ее, если это возможно. Это делается потому, что использование [`fseek`](fseek-fseeki64.md) и [`ftell`](ftell-ftelli64.md) для перемещения внутри файла может вызвать [`fseek`](fseek-fseeki64.md) неправильное поведение в конце файла. Не используйте **`t`** параметр, если требуется переносимость ANSI, так как это расширение Майкрософт.

Если параметр **`t`** или **`b`** не указан в *` mode`* , режим преобразования по умолчанию определяется глобальной переменной [`_fmode`](../../c-runtime-library/fmode.md) . Если **`t`** **`b`** аргумент или имеет префикс, функция завершается ошибкой и возвращает значение **`NULL`** .

Обсуждение текстового и двоичного режимов см. в разделе [Файловый ввод-вывод в текстовом и двоичном режиме](../../c-runtime-library/text-and-binary-mode-file-i-o.md).

## <a name="example"></a>Пример

```C
// crt_freopen_s.c
// This program reassigns stderr to the file
// named FREOPEN.OUT and writes a line to that file.

#include <stdio.h>
#include <stdlib.h>

FILE *stream;

int main( void )
{
   errno_t err;
   // Reassign "stderr" to "freopen.out":
   err = freopen_s( &stream, "freopen.out", "w", stderr );

   if( err != 0 )
      fprintf( stdout, "error on freopen\n" );
   else
   {
      fprintf( stdout, "successfully reassigned\n" ); 
      fflush( stdout );
      fprintf( stream, "This will go to the file 'freopen.out'\n" );
      fclose( stream );
   }
   system( "type freopen.out" );
}
```

```Output
successfully reassigned
This will go to the file 'freopen.out'
```
